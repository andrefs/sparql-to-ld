# Implementation Plan: sparql-to-ld

## Tech Stack (Confirmed)
- **Runtime:** Node.js LTS + TypeScript (strict mode)
- **HTTP Server:** Fastify
- **RDF Library:** n3
- **SPARQL Client:** sparql-http-client
- **Testing:** Vitest
- **Linting/Formatting:** ESLint + Prettier
- **Config Management:** dotenv, JSON config file

## Phase 2: Core Library Implementation

### 2.1 Type Definitions (`src/types/`)
- `Resource.ts`: `Iri`, `BlankNode`, `Literal`, `Triple`, `Dataset`, `ResourceDescriptor`
- `Config.ts`: `ServerConfig`, `SparqlEndpointConfig`, `RdfFormat`, `CorsOptions`
- `Errors.ts`: `SparqlToLdError`, `ConfigurationError`, `EndpointError`, `RdfParseError`

### 2.2 SPARQL Query Builder (`src/sparql/`)
- `query-builder.ts`:
  - `buildDescribeQuery(resourceIri: string, options?: DescribeOptions): string`
  - Support FedBox/CBD: include triples where resource is subject AND object
  - Parameterized to prevent injection
  - Add LIMIT and timeout hints
- `query-options.ts`: Define `DescribeOptions` interface (depth, includeInverse, propertyPaths)

### 2.3 RDF Handler (`src/rdf/`)
- `parser.ts`: `parseRdf(data: string, format: RdfFormat): Dataset`
- `serializer.ts`: `serializeDataset(dataset: Dataset, format: RdfFormat): string`
- `formats.ts`: Constants for accepted formats: `text/turtle`, `application/n-triples`, `application/ld+json`, `application/rdf+xml`
- Use n3 for parsing and serialization

### 2.4 SPARQL Client (`src/sparql/`)
- `client.ts`: `SparqlClient` class
  - Constructor takes endpoint URL, optional fetch options, timeout
  - `executeQuery(query: string): Promise<Dataset>`
  - Handles errors, timeouts, retries (maybe)
- Use sparql-http-client under the hood or custom wrapper

### 2.5 Configuration (`src/config/`)
- `loader.ts`: `loadConfig(options?: { env?: Record<string, string> }): Promise<ServerConfig>`
  - Load from `sparql-to-ld.json` (if exists)
  - Override with `.env` (using dotenv)
  - Override with `process.env`
  - Validate with Zod schema
- `types.ts`: Zod schemas for config validation

## Phase 3: HTTP Server Implementation (`src/server/`)

### 3.1 Main Server (`server.ts`)
- `createServer(config: ServerConfig): FastifyInstance`
- Register Fastify plugins:
  - `@fastify/cors` (if enabled)
  - `@fastify/helmet` (security)
  - `@fastify/rate-limit` (DoS protection)
- Set up logging with pino (or Fastify's built-in)
- Content negotiation middleware
- Error handling middleware

### 3.2 Routes (`routes/`)
- `resource-routes.ts`:
  - `GET /resource/:iri` (encode IRI properly)
    - Parse Accept header в†’ determine RDF format
    - Override with `?format=` query param if present
    - Fetch from SPARQL endpoint using SparqlClient
    - Return serialized RDF with proper Content-Type
    - Set caching headers (ETag, Last-Modified based on dataset hash)
    - Handle 404s, 500s appropriately
  - `GET /health` (health check)

---

## Phase 4: CLI Entry Point (`src/cli/`)

### 4.1 CLI Interface (`cli.ts`)
- Use `commander` or `yargs` for argument parsing
- Arguments:
  - `--config, -c`: Path to config file (default: ./sparql-to-ld.json)
  - `--port, -p`: Server port (default: 3000)
  - `--host`: Server host (default: 0.0.0.0)
  - `--env-file`: Path to .env file
- Load config, merge with CLI args
- Start server with `createServer`
- Handle shutdown signals (SIGINT, SIGTERM) gracefully

### 4.2 Main Entry (`index.ts`)
- Detect if running as CLI (`import.meta.main` or `require.main === module`)
- If CLI: call cli()
- If library: export public API (`createServer`, `loadConfig`, `SparqlClient`, etc.)
